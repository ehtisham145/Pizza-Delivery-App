3. The "Nuclear Option" (Force Sync with GitHub)

Use these three commands if you want to wipe your local changes and make your PC match GitHub exactly.
Step 1: git fetch origin main

    What it does: Checks GitHub for updates.

    The Result: It downloads the new data but does not change your local files yet. It just updates Git's internal records.

Step 2: git reset --hard origin/main

    What it does: This is the Force Sync command.

    The Result: It overwrites every file on your PC to match GitHub.

    Warning: Any code you wrote but did not push to GitHub will be permanently deleted.

Step 3: git clean -fd

    What it does: Removes "Untracked" files (files Git doesn't know about).

    The Result: * -f (force): Deletes extra files (like random .docx files).

        -d (directory): Deletes entire folders that are not in the Git repo.

4. Understanding Pydantic Annotations

When you use Annotated in Pydantic, the data goes through two main checks:
Step 1: The "Identity" Check (Data Type)

Pydantic acts as the first guard to check the "shape" of the data.

    If the type is wrong: (e.g., sending a "String" when it expects a "Float"), Pydantic stops the process immediately and raises an error.

    If the type is right: It lets the data pass to the next step.

Step 2: The "Background" Check (Your Function)

After the type is confirmed, the data is handed to your custom function (like an AfterValidator).

    The Goal: To check the logic (e.g., Is this a Pakistani phone number? Is the password strong?).

    Automatic Execution: You don't need to call the function manually; Pydantic runs it automatically.

5. Using git stash (The Temporary Locker)

Use this when you have unfinished work but need to download new updates from GitHub without losing your progress.
The Step-by-Step Process:

    Hide Your Work (git stash): Moves your current changes into a "Safe Box." Your folder becomes clean.

    Get New Code (git pull origin main): Downloads the latest updates from GitHub smoothly.

    Bring Back Your Work (git stash pop): Takes your work out of the "Locker" and mixes it with the new code.

When to use it?

    DO NOT use it for pushing/uploading code (use add, commit, push for that).

    DO use it when your code is half-finished and you aren't ready to commit, but you need the latest GitHub updates right now.

Pro Tip for Conflicts:

If git stash pop shows a "Conflict" error:

    Open the file in VS Code.

    Look for the highlighted lines where the code overlaps.

    Pick the version you want to keep and Save.
